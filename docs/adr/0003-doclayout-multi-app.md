# 0003. DocLayout をアプリ横断で再利用（app/hub frontmatter）

- 状態: Accepted
- 日付: 2026-07-06

## 背景

こうら日記に続き LinkMint の規約・プライバシーを追加するにあたり、規約ページのレイアウト
（`DocLayout.astro`）が「こうら日記」固定でハードコードされていた（タイトル接尾辞・パンくず・
戻りリンク）。アプリごとにレイアウトを複製すると重複が増える。

## 決定

- `DocLayout.astro` に frontmatter 由来の変数 `app` / `hub` を導入する。
  - `app`（アプリ名）… 未指定は `こうら日記`
  - `hub`（アプリのトップへのパス）… 未指定は `/koura-diary/`
- タイトル `{title} | {app}`、パンくず `{app} / {title}`、戻りリンク `← {app}トップにもどる` を
  これらで生成する。

## 理由

- 1 つのレイアウトを全アプリの規約ページで共有でき、重複を避けられる。
- **既定値をこうら日記にした**ことで、既存の `.md` は frontmatter 無変更のまま動く（後方互換）。
- 新規アプリは `.md` の frontmatter に `app` / `hub` を 2 行足すだけで正しい導線になる。

## 代替案

- **アプリごとに Layout を複製**: 重複・変更漏れの温床。却下。
- **ディレクトリから app を自動推定**: 暗黙的で読みにくく、表示名（「こうら日記」等）は
  パスから一意に決まらない。frontmatter 明示のほうが単純で確実。

## 影響

- 規約ページ追加の手順が「`.md` に layout + app/hub を指定」に定型化（SPEC F2）。
- タイトル・パンくずの文言を変えるときは DocLayout の 1 箇所を直せば全アプリに反映される。
