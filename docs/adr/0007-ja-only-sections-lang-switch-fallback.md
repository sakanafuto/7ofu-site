# 0007. ja 専用セクションの言語スイッチャ・hreflang フォールバック

- 状態: Accepted
- 日付: 2026-07-15

## 背景

adr/0004 は「新ページは ja/en 両方作る」を前提に、言語スイッチャと hreflang を
`alternatePath(pathname, lang)` で機械的に生成していた（`/en` 接頭辞を付け外しするだけ）。
その後ブログ（`/blog/`・`/blog/<slug>`）を **ja のみ**で追加したため、ブログページで:

- 言語スイッチャの EN ボタンが `/en/blog/...`（存在しない）へリンクし **404**
- hreflang が `<link hreflang="en" href=".../en/blog/...">` と存在しない URL を指し **SEO 不正**
  （Google の "alternate page missing" 要因）

nav は `lang === 'ja'` でブログ導線を出し分けていたが、スイッチャ／hreflang は対策漏れだった。

## 決定

- ja 専用セクションの単一情報源 `jaOnlyPrefixes`（`src/i18n/ui.ts`）を導入し、`hasAlternate(pathname)`
  で「対になる別 locale ページが存在するか」を判定する。
- 別 locale ページが無いページでは、**言語スイッチャは別 locale のトップへ逃がす**
  （ja 専用ブログの EN ボタン → `/en/`）。ボタン自体は残す。
- 別 locale ページが無いページでは、**hreflang の en alternate を出さない**（ja と x-default のみ）。

## 理由

- スイッチャを隠さずトップへ逃がすのは、英語サイトへの導線を残しつつ 404 を確実に無くすため
  （どのセクションが片 locale だけでも破綻しない汎用対策）。ユーザー承認済みの挙動。
- hreflang は「存在するページ同士の相互参照」が仕様。無い URL を宣言しない方が正しく、
  ja + x-default（=ja）だけで reciprocal 要件を満たす。
- `jaOnlyPrefixes` を単一情報源にすることで、en 版を用意した際に 1 箇所外せば全対策が解ける。

## 代替案

- **スイッチャを隠す**: nav のブログ出し分けと対称で一貫するが、ブログから英語サイトへ切り替える
  導線が消える（Home 等を経由する必要）。導線維持を優先して却下。
- **en 版ブログを用意する**: ブログ本文の英訳コストが高く、目的（App Store 審査・海外導線）に
  ブログは含まれない。見送り。
- **nav と共通の単一判定関数に完全統一**: nav は「リンクを出すか」、スイッチャは「現在ページに対を
  持つか」で述語が異なる。今回は相互参照コメントで対応し、過度な抽象化は避けた。

## 影響

- ja 専用にしたいセクションは `jaOnlyPrefixes` に接頭辞（末尾スラッシュ無し）を登録する。
  スイッチャ・hreflang が自動でフォールバックする。en 版を用意したら登録を外す。
- `jaOnlyPrefixes` と nav（BaseLayout.astro）のブログ出し分けは同じ事実の二重管理。
  片方だけ変えないよう両所に相互参照コメントを置いた（en 版ブログを出す際は両方を直す）。
- adr/0004「未対応 locale があるとスイッチャがリンク切れ」の制約を、登録済みセクションに限り緩和する。
