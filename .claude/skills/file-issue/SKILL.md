# Issue を発行（/file-issue <内容>）

ユーザーがやりたいことを構造化した Issue にして起票する skill。

## 手順

### 1. 内容を整える
ユーザーの要旨から、明確な**タイトル**と構造化した**本文**を作る（日本語）。本文テンプレ:
```markdown
## 背景 / 目的
…（なぜ・何のため）

## やりたいこと
…（具体的に。画面・データ・挙動）

## 受入条件
- [ ] …
- [ ] …

## 補足（任意）
…（設計メモ・参考・スコープ外）
```
- 既存の SPEC（docs/SPEC.md）の機能 ID（F2 等）があれば本文で参照する。

### 2. ラベル（任意）
```sh
gh label list
```
- 内容に合うラベルがあれば付ける（`bug` / `enhancement` / `documentation` 等）。無理に作らない。

### 3. 起票
```sh
gh issue create --title "…" --body "…" [--label enhancement]
```
- `gh` はキーチェーン検証でサンドボックスに弾かれることがある → サンドボックス無効で再実行。

### 4. 報告
- 発行した Issue 番号・URL を返す。複数依頼なら 1 件ずつ起票して列挙。

## 注意
- 機密・個人情報を本文に書かない。
- ユーザーの意図が曖昧なら、起票前に 1 点だけ確認してから整える（粗い Issue を量産しない）。
- 起票だけ。実装はしない（実装は [/do-issue](../do-issue/SKILL.md)）。
